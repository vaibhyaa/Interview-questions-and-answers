482. How do you decide between Context, Redux Toolkit, and React Query?
Context API → small/global state (theme, auth)
Redux Toolkit → complex client state (forms, UI logic, large apps)
React Query → server state (API data, caching, async)
Context/Redux = client state, React Query = server state


483. How do you separate server state from client state?
server state:
Comes from API
Needs caching, refetching

👉 Client state:
UI state (modals, filters, inputs)

👉 Practice:
Use React Query for server data
Use Context/Redux for UI state
