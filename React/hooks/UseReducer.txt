
I use useState for simple independent state and useReducer when state becomes more complex, related, or transition-driven. useReducer is better when multiple values change together, there are many action types, or business logic needs to be centralized. It gives a predictable event-driven flow: dispatch an action, reducer calculates the next state. It also works very well with Context API for shared feature-level state.

What is useReducer?
useReducer is a React Hook used to manage state with a reducer function.
useReducer is a Hook for managing complex state in a predictable, action-based way.

190. What are action, dispatch, reducer?
An action is a plain object that describes what happened. 
{ type: "increment" }
{ type: "setName", payload: "Vaibhav" }
Usually contains:
type → what event occurred
payload → optional data
dispatch is the function used to send an action to the reducer.
dispatch({ type: "increment" });
dispatch({ type: "setName", payload: "Vaibhav" });
A reducer is a function that receives:
current state
action
and returns:
next state
function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { ...state, count: state.count + 1 };
    default:
      return state;
  }
}

Flow:
UI event → dispatch(action) → reducer(state, action) → new state → re-render
An action describes what happened, dispatch sends that action, and the reducer decides the next state based on current state and action.




191) Why should reducers be pure functions?
Reducers should be pure because React expects state updates to be predictable, testable, and side-effect free.


192) Can reducers contain async logic?
No — reducers should not contain async logic.
Reducers decide state transitions


193)  How do you structure reducer actions in large apps?
In large apps, actions should be consistent, scalable, and easy to maintain.
